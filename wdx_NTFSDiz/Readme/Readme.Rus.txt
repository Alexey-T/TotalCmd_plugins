NTFS Descriptions
Content-плагин для Total Commander 7.0+
Автор: Алексей Фомин (ledsoft@tut.by)

Описание
--------
Плагин предназначен для просмотра, создания и редактирования комментариев к файлам и каталогам,
причём эти комментарии хранятся в виде потоков в файловой системе NTFS.

Преимущества такого способа:
-не создаются лишние файлы в каждом каталоге (типа descript.ion или files.bbs);
-комментарии сохраняются даже после копирования проводником;
-вы можете создавать сколько угодно комментариев с разными именами (в разных потоках).

Недостатки:
-потоки можно создавать только на дисках с файловой системой NTFS,
 соответственно требуется Windows NT, 2000, XP, Vista и т.п.
 При копировании файла/каталога на диск FAT32 комментарии(потоки) будут утеряны!

Использование
-------------
Имена потоков, используемых для хранения комментариев можно задавать в файле NTFS_diz.ini (максимум 10).
По умолчанию там одно имя (Comment). Заданные имена отображаются в меню выбора полей плагина.

Несколько различных комментариев можно использовать, например:
-для пометки некоторых важных файлов (а потом перейти к ним, используя поиск с плагинами);
-для раскраски определённых каталогов в другой цвет (задать в комментарии значения red, green и т.д. и настроить раскраску по шаблону);
-помечать книги, музыку, фильмы (интересно/отстой, смотрел/не смотрел и т.д.);
-и многое другое...
-и всё это одновременно :)

Для просмотра комментариев создайте пользовательские колонки и выберите плагин ntfs_diz.
Для создания/редактирования комментариев вызовите диалог смены атрибутов и опять же выберите плагин.

Не забывайте, что вы легко можете перенести свои комментарии из descript.ion в NTFS-потоки и обратно,
используя диалог смены атрибутов!

Для всего этого нужен Total Commander версии 7.0 или выше!
Только для просмотра (не редактирования) достаточно версии 6.5.

Если Total Commander не копирует потоки, установите в вашем wincmd.ini: [Configuration] CopyStreams=1

По умолчанию, при изменении комментария обновляется дата файла.
Если вас это не устраивает, измените в NTFS_diz.ini в секции [Options] параметр KeepTime=1
Тогда дата будет оставаться прежней, но ТС не будет автоматически обновлять содержимое панелей.
Для ручного обновления перейдите в другой каталог или на другую вкладку и вернитесь обратно.
